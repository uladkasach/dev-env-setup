# playtest: git backup

## prerequisites

- aws cli configured with s3 write access
- s3 bucket exists (e.g., `s3://vlad-backups/`)
- bash_aliases.sh sourced (run `source ~/.bash_aliases` or restart terminal)
- ~/git/ directory exists with at least one repo
- optional: ~/.claude/ directory for full test

---

## sandbox

this playtest operates on:
- real ~/git/ and ~/.claude/ directories (read-only for tar)
- real s3 bucket (writes timestamped archives)
- no file modifications to repo or home directory

cleanup: archives uploaded to s3 can be deleted after test via `aws s3 rm`

---

## happy paths

### play.1 = backup all repos with claude sessions

**setup:**
```sh
# ensure ~/git/ exists
ls ~/git/

# ensure ~/.claude/ exists
ls ~/.claude/
```

**action:**
```sh
git backup --repo all --into s3://vlad-backups/machine=vlad.laptop1
```

**expected outcome:**
- displays repo tree with 🏔️ emoji headers for each repo
- displays branch tree with 🌲 emoji for each worktree
- displays status for each worktree (✋ unstaged, 👌 merged, etc.)
- creates and uploads tar.gz archives
- displays `🍯 got em` confirmation header
- displays s3 uri with `effectiveAt=$timestamp` in path
- displays file sizes for git.tar.gz and .claude.tar.gz

**pass if:**
- output contains `🍯 got em`
- output shows two s3 uris (git.tar.gz and .claude.tar.gz)
- `aws s3 ls s3://vlad-backups/machine=vlad.laptop1/` shows new effectiveAt directory

**fail if:**
- command exits with error
- no `🍯 got em` in output
- s3 uris not displayed

---

### play.2 = verify s3 path structure

**action:**
```sh
# after play.1, list the backup directory
aws s3 ls s3://vlad-backups/machine=vlad.laptop1/ --recursive | head -20
```

**expected outcome:**
- path follows pattern: `machine=vlad.laptop1/effectiveAt=YYYY-MM-DDTHH:MM:SSZ/~/`
- contains `git.tar.gz`
- contains `.claude.tar.gz` (if ~/.claude/ existed)

**pass if:**
- effectiveAt timestamp is ISO8601 format
- both tar.gz files present under `~/` prefix

**fail if:**
- timestamp format differs
- files not under `~/` prefix

---

### play.3 = help output

**action:**
```sh
git backup --help
```

**expected outcome:**
- displays usage information
- shows --repo and --into options
- exits cleanly (exit code 0)

**pass if:**
- output contains `--repo` and `--into`

**fail if:**
- error message instead of help

## edgey paths

### edge.1 = absent --repo flag

**action:**
```sh
git backup --into s3://vlad-backups/machine=test
```

**expected outcome:**
- displays `error: --repo is required`
- exits with non-zero status

**pass if:**
- error message mentions --repo
- exit code is non-zero (`echo $?` shows 1)

**fail if:**
- command proceeds without --repo

---

### edge.2 = absent --into flag

**action:**
```sh
git backup --repo all
```

**expected outcome:**
- displays `error: --into is required`
- exits with non-zero status

**pass if:**
- error message mentions --into
- exit code is non-zero

**fail if:**
- command proceeds without --into

---

### edge.3 = invalid s3 uri (no s3:// prefix)

**action:**
```sh
git backup --repo all --into bucket/path
```

**expected outcome:**
- displays `error: --into must start with s3://`
- exits with non-zero status

**pass if:**
- error message about s3:// prefix
- exit code is non-zero

**fail if:**
- command attempts upload to malformed uri

---

### edge.4 = absent ~/.claude/ directory

**action:**
```sh
# temporarily rename .claude to simulate absence
mv ~/.claude ~/.claude.bak

# run backup
git backup --repo all --into s3://vlad-backups/machine=test-no-claude

# restore .claude
mv ~/.claude.bak ~/.claude
```

**expected outcome:**
- displays warn about absent ~/.claude/
- creates and uploads git.tar.gz only
- displays `🍯 got em` with single s3 uri
- does not fail

**pass if:**
- warn mentions ~/.claude/ not found
- git.tar.gz still uploaded
- output shows single file (not two)

**fail if:**
- command fails due to absent .claude
- no warn displayed

---

### edge.5 = absent ~/git/ directory

**action:**
```sh
# temporarily rename git to simulate absence
mv ~/git ~/git.bak

# run backup
git backup --repo all --into s3://vlad-backups/machine=test-no-git

# restore git
mv ~/git.bak ~/git
```

**expected outcome:**
- displays `⛈️  no git dir found` message
- exits with non-zero status
- no upload attempted

**pass if:**
- output contains "no git dir found"
- exit code is non-zero

**fail if:**
- command proceeds without ~/git/

---

### edge.6 = invalid --repo value

**action:**
```sh
git backup --repo this --into s3://vlad-backups/machine=test
```

**expected outcome:**
- displays error about invalid --repo value
- only `all` is supported in v1

**pass if:**
- error message indicates only `all` is supported

**fail if:**
- command attempts backup with `this`

---

## pass/fail criteria

### overall pass

✓ all happy paths complete without error
✓ all edge paths produce expected error behavior
✓ s3 archives are accessible and contain expected structure
✓ output format matches vision (emojis, tree structure)

### overall fail

✗ any happy path fails
✗ any edge path proceeds when it should error
✗ s3 archives absent or malformed
✗ output lacks expected elements (🏔️, 🌲, 🍯)
