# handoff: termwork kitty tabs ‚Äî one window, N tabs via `--tab`

## .what

let one kitty window host multiple ducts as tabs. address a tab with a
`--tab N` qualifier on the extant verbs. `--tab 0` = default (today's
behavior), so every current call site keeps working unchanged.

target supervisor view:

```
1 kitty window per worktree (tree)
  ‚îú‚îÄ tab 0: mechanic  (attached to $tree/mechanic tmux)
  ‚îî‚îÄ tab 1: foreman   (attached to $tree/foreman  tmux)
```

## .why

- one window per tree = clean supervisor view; alt-1 / alt-2 flip mechanic‚Üîforeman
- fewer stray windows (a driver runs 5-8 trees = 10-16 windows today)
- mechanic + foreman are two views of ONE unit of work; they belong together
- `--tab` as a qualifier (not a new verb) keeps the surface small and every
  prior caller untouched

## .design: `--tab N` on every verb, default 0

do NOT add a new verb. extend the current ones:

| verb | today | with tabs |
|------|-------|-----------|
| `term.open` | spawns a window for a duct | `--tab 0` spawns window; `--tab 1` adds a tab to the extant window for that tree |
| `term.read` | reads the window | `--tab N` reads tab N |
| `term.send` | sends to the window | `--tab N` sends to tab N |
| `term.stop` | closes the window | `--tab N` closes tab N; closing the last tab closes the window |
| `term.list` | lists windows | lists windows with tabs nested |

`--tab` defaults to `0` everywhere ‚Üí 100% backward compatible.

### open semantics

`term.open --via kitty --on $duct --tab N`:

- **tab 0** (default): findsert the OS window for this duct (current behavior)
- **tab >0**: the window must already exist (opened at tab 0). add a tab to
  THAT window, attach it to `$duct`'s tmux session. if the host window is
  absent ‚Üí fail clear (hint: open `--tab 0` first).

open questions for termwork to decide:
- how does `--tab 1` know WHICH window to attach into? two options:
  1. **explicit host**: `--tab 1 --into $tree/mechanic` (name the host duct)
  2. **shared tree prefix**: infer host from a shared slug prefix
     (`$tree/mechanic` and `$tree/foreman` share `$tree`)
  - lean option 1 (explicit) ‚Äî no magic, no prefix parsing.

## .the mechanism (verified)

every window `term.open` spawns already sets:
- `allow_remote_control=yes`
- `listen_on=unix:/tmp/kitty-{kitty_pid}`

so a tab drops into that same window over its socket:

```bash
kitten @ --to "$socket" launch \
  --type=tab \
  --tab-title "foreman" \
  --cwd "$cwd" \
  -- "$shell" -l -i -c "tmux attach-session -t '$duct_session'"
```

`--to <socket>` targets that one kitty instance (= one OS window);
`--type=tab` adds the tab inside it. `launch` prints the new kitty window id
(the tab's content window) on stdout ‚Äî capture it to address the tab later.

per-tab targeting for read/send/stop uses `--match`:

```bash
kitten @ --to "$socket" --match "id:$tabWindowId" get-text     # read
kitten @ --to "$socket" --match "id:$tabWindowId" send-text ...  # send
kitten @ --to "$socket" --match "id:$tabWindowId" close-window   # stop tab
```

## .registry changes (the real work)

`~/.termwork/$pid.json` is keyed one-window-per-duct today. tabs break that
1:1 ‚Äî one window (pid/socket) now hosts N ducts.

suggested record shape:

```json
{
  "pid": 12345,
  "socket": "unix:/tmp/kitty-12345",
  "tabs": [
    { "tab": 0, "duct": "tree.mechanic", "host": "", "kittyId": 1, "title": "mechanic" },
    { "tab": 1, "duct": "tree.foreman",  "host": "", "kittyId": 2, "title": "foreman"  }
  ],
  "startedAt": 1751500000000
}
```

changes to the operations in `src/termwork.sh`:

- `__term_register` ‚Äî register a tab (append to `tabs[]`) vs a new window
- `__term_find_by_duct` ‚Äî must find a duct whether it is tab 0 (the window)
  or a tab inside a window; return `(socket, kittyId)` so callers can target it
- `term.open` ‚Äî branch on `--tab`: 0 = spawn window (as now); >0 = `launch
  --type=tab` into the host window's socket, capture kittyId, register the tab
- `term.read` / `term.send` / `term.stop` ‚Äî take `--tab N`, look up the
  kittyId, use `kitten @ --match "id:$kittyId"`; `term.stop` on the last tab
  closes the window (prior close-window path)
- `term.list` ‚Äî nest tabs under their window

## .idempotency

- `term.open --tab 1` findserts: if a tab for `$duct` already lives in the
  host window, focus it (no dup)
- host window absent for `--tab >0` ‚Üí fail clear

## .caller integration (nheuron side ‚Äî not termwork's job)

once `--tab` ships, nheuron's `git.tree.duct` adds one line after opening the
mechanic window:

```bash
term.open --via kitty --on $slug_mechanic                          # tab 0 (window)
term.open --via kitty --on $slug_foreman --tab 1 --into $slug_mechanic  # tab 1
```

no action needed from termwork beyond `--tab` support. nheuron owns the wiring.

## .acceptance

- `term.open --on B --tab 1 --into A` adds a 2nd tab to A's window, attached to B's tmux
- alt-1 / alt-2 flip between the two tabs
- `term.read --on B --tab 1` reads the foreman tab (not the mechanic tab)
- `term.send --on B --tab 1 --what ...` sends to the foreman tab
- `term.stop --on B --tab 1` closes only the foreman tab; window + tab 0 stay
- `term.stop` on the last tab closes the window
- `term.list` shows the window with both tabs nested
- every prior call site (no `--tab`) behaves exactly as before (= tab 0)
- idempotent: re-run of `--tab 1` focuses the extant tab, no dup
