# vision: nimg — vim-style image viewer

## the outcome world

### before/after contrast

| before | after |
|--------|-------|
| open image in eog/feh, reach for mouse to pan | `nimg screenshot.png`, h/j/k/l to navigate |
| zoom with scroll wheel or trackpad gestures | ctrl+j/k to zoom out/in |
| arrow keys for pan (distant from home row) | vim muscle memory just works |
| separate mental model for image viewing | same hjkl you use everywhere else |

### day-in-the-life

you're reviewing a long screenshot of a webpage. instead of:
1. open in default viewer
2. grab mouse
3. scroll around
4. squint, zoom with trackpad

you:
```sh
nimg ~/screenshots/ticket-1234.png
```
then hjkl to pan, ctrl+k to zoom in on the detail, `G` to jump to bottom, `q` to quit. hands never leave home row.

### the "aha" moment

first time you instinctively hit `j` to pan down an image and it works — you realize your vim muscle memory now works for images too.

## user experience

### usecases

| goal | command | flow |
|------|---------|------|
| view single image | `nimg image.png` | opens at 100% zoom, hjkl to pan |
| browse folder | `nimg *.png` | space/backspace to navigate, hjkl within each |
| inspect diagram detail | ctrl+k to zoom in, hjkl to find spot, ctrl+j to zoom out |
| review long screenshot | `G` to jump to bottom, `gg` to top |

### contract

**input:** `nimg <path>...`
- one or more image paths (globs supported)

**output:**
- pqiv window with vim keybindings pre-configured

### timeline

1. human runs `sync.devenv` (or just `sync.devenv.bashaliases`)
2. alias `nimg` is available
3. first run: if pqiv not installed, prompts to run install procedure
4. config file `~/.config/pqivrc` created by install procedure (XDG-compliant path)

## evaluation

### how well does it solve the goals?

| goal | solved? | notes |
|------|---------|-------|
| vim-style navigation | yes | hjkl, gg/G all work |
| keyboard-centric zoom | yes | ctrl+j/k |
| minimal friction | yes | single alias, familiar keys |
| wayland support | yes | works natively on wayland |

### pros

- zero learning curve for vim users
- lightweight (pqiv is ~330KB)
- config is human-readable (key { action } syntax in [sections])
- works on both X11 and wayland

### cons

- pqiv not in all distros (debian/ubuntu: yes)
- no auto-center on startup (pqiv limitation)
- `gg` requires fast double-tap (pqiv interprets key sequences)

### edgecases & pit of success

| edgecase | approach |
|----------|----------|
| pqiv not installed | alias checks and prompts |
| config absent | install procedure creates it |

## groundwork

### external research

**pqiv documentation (verified via web search + github):**
- pqiv is in debian/ubuntu repos: `sudo apt install pqiv` (version 2.13.3-1)
- keybinding format is `key { action }` in config files
- config path priority: `~/.config/pqivrc` > `~/.pqivrc` > `/etc/xdg/pqivrc` > `/etc/pqivrc`
- **decision:** use `~/.config/pqivrc` (XDG-compliant, highest priority)
- multi-key sequences supported with 0.5s timeout (e.g., `gg` works)
- all actions verified: `shift_x`, `shift_y`, `set_shift_align_corner`, `set_scale_level_relative`
- sources:
  - https://github.com/phillipberndt/pqiv
  - https://packages.debian.org/pqiv
  - https://manpages.ubuntu.com/manpages/focal/man1/pqiv.1.html

**wayland support (verified via local test):**
- NO separate `pqiv-wayland` binary — pqiv auto-detects GTK backend
- tested on cosmic wayland: works without issues
- no workaround needed

### internal research

**bash_aliases.sh patterns:**
- aliases are simple: `alias name='command'`
- functions for complex logic: `function name { ... }`
- file synced via `sync.devenv.bashaliases` to `~/.bash_aliases`
- location: `src/bash_aliases.sh`

**install procedure patterns:**
- procedures in `src/install_env.pt*.sh`
- naming: `install_*` for apt/setup, `configure_*` for config files
- invoked manually: `source src/install_env.pt*.sh && install_*`
- location: follow brief `.agent/repo=.this/role=any/briefs/rule.require.install-via-procedures.md`

**relevant extant patterns:**
- `src/bash_aliases.sh:2` — `alias vim='nvim'` (simple alias)
- `src/install_env.pt4.terminal.sh` — `install_neovim()`, `configure_neovim()`

## what is awkward?

1. **gg double-tap timing** — pqiv's key sequence parse has 0.5s timeout. if user types `g` then pauses >0.5s, it interprets as single `g`. this is a pqiv behavior, not something we can fix.

2. **100% zoom position** — pqiv opens at 100% zoom but doesn't center. user may need to pan to see the image. this is noted in the wish as a limitation we accept.

3. **install vs alias** — the alias is useless without pqiv+config. two-step setup (install procedure + sync aliases) is standard for this repo, but new users might just try `nimg` and get confused. the alias can check for pqiv and emit a helpful error message.
